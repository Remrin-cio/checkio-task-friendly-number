大きい数は見やすくしたいので、そういうコードを書いてみましょう。

いくつかのルールに基づいてnumberを文字列に変換する関数を書いてください。 まずは与えられた(base 引数; デフォルトは 1000)で割ります。 商の値は小数で、小数点以下の桁数は (decimals 引数; デフォルトは 0)です。 decimal = 0ならば、(5.6⇒5, -5.6⇒-5)のように値を原点方向に丸めます, それ以外は値を四捨五入します。 decimalsが小数点以下の桁数より大きい場合は、ゼロをつなげてください。 桁を表すアルファベットを付けて数を表します。 桁を表すアルファベットのリストは(powers 引数; デフォルト ['', 'k', 'M', 'G', 'T', 'P', 'E', 'Z', 'Y']). 接尾辞として (suffix 引数; デフォルトは ‘’)が与えられたら、その接尾辞を後ろに付けてください。 powersが不足している場合は、与えられたpowerのうち最大のものを使います。 値がゼロのときは桁を表すアルファベットは付けませんが、接尾辞は付けます。

例を見るとわかりやすいです。

  • n=102
    結果: "102", baseはデフォルトの1000で、102はそれよりも小さいです。
  • n=10240
    結果: "10k", baseはデフォルトの1000で、値は丸めます。
  • n=12341234, decimals=1
    結果: "12.3M", 小数点以下が1桁です。
  • n=12000000, decimals=3
    結果: "12.000M", ゼロがつながります。
  • n=12461, decimals=1
    結果: "12.5k", 四捨五入です。
  • n=1024000000, base=1024, suffix='iB'
    結果: '976MiB', baseがデフォルトの1000とは異なり、suffixも与えられています。
  • n=-150, base=100, powers=['', 'd', 'D']
    結果: '-1d', 負の数が与えられ、原点方向に丸めます。
  • n=-155, base=100, decimals=1, powers=['', 'd', 'D']
    結果: '-1.6d', 負の数が与えられ、四捨五入をします。
  • n=255000000000, powers=['', 'k', 'M']
    結果: '255000M', 充分なpowersが与えられていません。

入力: 整数値。 キーワード引数「base」も整数で、デフォルトは1000。 キーワード引数「decimals」は整数で、デフォルトは0。 キーワード引数「powers」は文字列のリストで、デフォルトは['', 'k', 'M', 'G', 'T', 'P', 'E', 'Z', 'Y']。

出力: 文字列型に変換した数字

応用例:物理学やＩＴの世界では様々な数を扱います。 単純で読みやすい数字で表したほうがよい場合があります。ギガバイトについて扱う時にはバイトやキロバイトについては正確に表す必要がないこともあります。

条件: 1 < base ≤ 1032
-1032 < number ≤ 1032
0 ≤ decimals ≤ 15
